# ZTE-20260716-0001 — Agent-operated production hardening

## Objective

Extend the production-grade MACS Digital Media public site with a narrow, authenticated, observable site-operations surface that Agent MAXX can use without exposing unrestricted execution, provider credentials, real client records, or production mission state.

This sprint does not modify `executiveusa/macs-agent-portal`. It preserves the repository boundary in `AGENTS.md`, `README.md`, and `CONTEXT.md`.

## Context synthesized

- PR #7 is merged to `main` at `91ef2654b163018e79407cde8d04789874293b46`.
- The public site owns marketing, content, applications, safe synthetic demonstrations, SEO, and public endpoints.
- Unrestricted agent execution remains in `macs-agent-portal`.
- The self-hosted Supabase stack on Hostinger is the persistence target.
- Existing patterns use server-only Supabase access, forced RLS, manual migrations, Vercel deployment, Playwright, localized copy, reduced-motion support, and no secrets in GitHub.
- The project vault contains the required provider categories, but values will never be copied, logged, or committed.

## Vertical outcome

Agent MAXX receives a safe machine-readable contract and authenticated API for managing a small allowlisted set of public content blocks. The website remains fully functional when the agent API, Supabase, or managed content is unavailable.

## Validation criteria

Binary pass requirements:

1. `/.well-known/agent.json` returns a versioned public capability manifest with no secrets.
2. `/api/agent/status` rejects missing or invalid bearer credentials with `401` and `Cache-Control: no-store`.
3. `/api/agent/content` rejects unauthorized reads and writes.
4. Writes require an idempotency key and accept only allowlisted content keys, locales, fields, lengths, and safe internal/HTTPS links.
5. The public homepage renders normally when no managed announcement exists or Supabase is unavailable.
6. When an enabled managed announcement exists, it renders with localized content and a clear optional CTA.
7. Database tables force RLS and revoke `anon` and `authenticated` privileges.
8. Unit coverage for the authentication and content-contract modules is at least 80% statements/branches/functions/lines.
9. Lint, typecheck, production build, unit coverage, Playwright, screenshot generation, CodeRabbit, and Vercel preview pass.
10. No secret values appear in source, logs, screenshots, reports, or API responses.

## Rollback

- Revert the eventual squash merge.
- Remove `SITE_AGENT_TOKEN` from Vercel to disable authenticated operations immediately.
- Leave database tables in place during rollback to preserve audit history; destructive rollback requires a separate reviewed migration.
- Managed content is optional and failure-isolated, so the existing homepage remains the fallback.
