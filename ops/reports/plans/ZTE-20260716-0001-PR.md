Add an authenticated, observable, reversible Agent MAXX site-operations surface to the production public website without crossing the repository boundary into unrestricted agent execution.

- Constant-time bearer authentication
- Authenticated site status endpoint

- Idempotency-key requirements and bounded agent identity
- Bilingual optional homepage announcement

- 80% minimum line, branch, and function coverage gates for critical modules
- Playwright fail-closed endpoint tests
- Agent operations runbook and refreshed repository documentation

## Protected boundaries

- No unrestricted execution
- No real client-record access
- No provider credentials
- No browser-control access
- No production mission state
- No changes to `executiveusa/macs-agent-portal`

## Verification

CI must pass lint, typecheck, unit coverage, production build, Chromium tests, screenshot generation, code review, and Vercel preview.

Authenticated preview smoke testing requires `SITE_AGENT_TOKEN` and the Supabase migration in the preview environment. Production migration and merge remain blocked by the explicit Bambu production gate.

## Rollback

- Revert the squash merge and redeploy the prior production commit.
- Rotate or remove `SITE_AGENT_TOKEN` for immediate operations shutdown.
- Preserve audit tables unless a separately approved destructive migration is executed.
